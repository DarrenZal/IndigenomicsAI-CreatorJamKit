# Contributing

This repo supports the Creator Jam. Contributions can be specs, sketches, code, examples, prompts, commitments, refusals, review notes, or witness rollups.

## Contribution Rules

Use public, sample, or explicitly display-approved material only.

Do not add:

- private contact details
- private participant notes
- credentials, tokens, or `.env` files
- protected cultural, linguistic, ceremonial, or Nation-specific material
- authority-bound material that has not been approved for this exact use
- real participant receipts unless display approval is explicit

## How To Contribute

If you use GitHub:

1. Create a branch.
2. Add or edit files under `docs/`, `templates/`, `examples/`, or `workshop/`.
3. Run the local checks in `README.md`.
4. Open a pull request.

If you do not use GitHub:

1. Answer the quick card questions.
2. A facilitator can enter your offering.
3. You can review the entered text before it is used, displayed, or bundled.

## Review States

Use these states plainly:

- `public_sample`: safe to use as public sample material
- `display_approved`: contributor approved display
- `review_required`: do not use until the named review route clears it
- `protect_and_preserve`: record the boundary, not the protected content
- `do_not_use`: do not use, bundle, display, summarize, or send to AI

## Pull-Back To IndigenomicsAI

Some work here may later be invited back into the internal IndigenomicsAI repo.

Before that happens:

1. Confirm contributor permission.
2. Preserve attribution.
3. Preserve refusal boundaries.
4. Record unresolved review requirements.
5. Copy only the approved artifact, not private discussion or protected context.
