# Offering Integration Session

## Purpose

Help offerings meet without forcing them into one mega-spec.

## Flow

1. Use the quick card for each offering.
2. Record permission and review state.
3. Name what fits.
4. Name what does not fit yet.
5. Preserve refusal and protected boundaries.
6. Form candidate bundles.
7. Select at most one bundle for a build attempt.
8. Freeze the selected bundle into build attempt instructions.

## Outputs

- fragment inventory
- candidate bundles
- review queue
- does-not-fit-yet list
- selected bundle
- frozen handoff packet
