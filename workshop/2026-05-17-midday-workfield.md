## Next Build Directions

These are not a priority order. Pick based on what kind of energy we want next.

1. **Build a tiny composition engine prototype.**
   Read fixture JSON, list candidate records, show transitions, run hard checks, and emit a draft composition report. This tests whether the docs are becoming executable.

2. **Write the conversation-to-records hazard map.**
   Do not build transcript inference yet. Name exactly how speech, jokes, questions, dreams, and tentative offers can be over-extracted into claims or commitments.

3. **Create the app gateway sidecar fixture.**
   Show the Indigenomics AI web app path from participant entry to offering card to composition suggestion to reviewer receipt.

4. **Map the witnessing architecture.**
   Clarify software receipts, human review, facilitator notes, contributor acknowledgement, protocol authority, display review, and Waka-style witness concepts without collapsing them.

5. **Extend commitment pool toward flow funding.**
   Take the accepted fixture commitment and test what can or cannot become a flow-funding edge. Keep non-underwriting, non-ranking, and withdrawal boundaries explicit.

6. **Create a coordination map layers spec.**
   Treat offers, needs, commitments, refusals, issues, specs, places, evidence, and pathways as map layers with visibility and authority boundaries.

7. **Build an agentic coding harness over the kit.**
   Give an agent a selected bundle, let it attempt a small build, then force reviewer/witness receipt output.

## Likely Best Next Move

If the goal is to turn the theory into working machinery today, build the tiny composition engine prototype.

If the goal is to protect against the highest-risk future workflow, write the conversation-to-records hazard map.

If the goal is to feed the Indigenomics AI web app, create the app gateway sidecar fixture.
