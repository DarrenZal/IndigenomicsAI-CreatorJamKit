# Discourse: Open Kit Collective Demo

## Proposal

Use the Open Kit Collective Demo as the selected execution bundle for Composition v0.

## Arguments For

- It can use public sample material and display-approved commitments.
- It already has a manual/static Open Kit demo path.
- It exercises the whole flow: offering integration, frozen build instructions, builder/reviewer/witness notes, and receipt rollup.
- It keeps TELUS/Jupyter optional rather than required.

## Arguments Against

- It is still a rehearsal, not a real participant bundle.
- The static HTML demo has placeholder content hashes that need repair before reuse beyond sample rehearsal.
- It does not yet show the fuller story/receipt wall experience.

## Open Questions

- Should the placeholder content hashes in the static HTML demo be replaced with deterministic sample hashes?
- What should be shown publicly if real participants do not approve receipt display?
- How much of the TELUS harness output should be shown to non-technical participants?

## Decision

Selected for the Composition v0 build attempt. Keep it narrow: public/sample material, manual/static path first, TELUS notes-only harness as optional support, and no protected or authority-bound material in AI prompts.
