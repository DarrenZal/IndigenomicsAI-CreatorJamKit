# AI Attempt Review Pattern

This sample shows how to review an AI-assisted build attempt without treating AI output as authority.

## Roles

- Builder: attempts the packet using authorized inputs.
- Reviewer: checks acceptance criteria and refusal boundaries.
- Witness: records what happened, including failures and not-run items.

## Boundaries

- Protected content is not sent to AI.
- AI does not decide consent or cultural authority.
- Reviewer checks the run; reviewer does not certify legitimacy.
- Witness records what happened; witness does not convert output into approval.
