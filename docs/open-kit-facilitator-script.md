# Open Kit Facilitator Script - v0

## Purpose

Use this for the narrow first public Open Kit demo:

- commitment-pool primer
- witnessing receipt schema v0.2
- sample receipt
- manual/static rendering
- optional TELUS/Jupyter showcase only if ready

Do not expand the demo into participant gateway, full multi-witness runtime, Waka bridge, or knowledge commons backend.

## Opening

This is a learning and building demo. We are not launching a production pool, and we are not asking anyone to accept a finished system.

We are going to witness a simple pattern:

1. A person makes a public ecological commitment.
2. The commitment becomes visible in a small local demo.
3. The system shows how that commitment can be recorded as a receipt.
4. We name what worked, what was partial, what was refused, and what remains outside this run.

## What This Is

This is a worked example of how offerings can become commitments, how commitments can be witnessed, and how receipts can help track contribution, attribution, permission, refusal, and repair.

The example uses public ecological demo content. It does not use cultural, linguistic, ceremonial, Nation-specific, participant-private, or protected research material.

Review place/territory acknowledgment wording before projecting the sample receipt publicly.

## What Receipts Mean

Receipts record what happened:

- who contributed
- what was contributed
- what the stated purpose was
- what permission or review state was declared
- what refusal or withdrawal path exists
- what was witnessed or checked

Receipts do not establish legitimacy, authority, or certification. A receipt is a record of contribution and witnessing. It does not replace human, community, Nation, steward, or ceremony authority.

## Protected Material Boundary

Protected or authority-bound content should not be processed by AI for checking.

If something is private, ceremonial, linguistic, Nation-specific, participant-private, steward-held, or otherwise review-required, the correct action is to mark review required and preserve it outside the AI run.

The system can record that review is required. It cannot compute legitimacy.

## TELUS / Jupyter Boundary

TELUS-hosted models or a TELUS/Jupyter notebook may be used as an optional showcase path if ready.

Manual witnessing is valid. If TELUS/Jupyter is not ready, the Open Kit demo still works as a local/static walkthrough with a receipt example.

Runtime failure is not bundle failure. If a model, notebook, or endpoint fails, we name that clearly and continue with manual witnessing.

## Demo Steps

1. Open the local commitment-pool primer. If your browser blocks opening the HTML directly, serve the repo locally and open the same file through localhost.
2. Say: "This is a worked example, not a production pool."
3. Enter one public ecological commitment.
4. Show the commitment tuple.
5. Show the v0.2-style receipt fields or sample receipt v0.2. If validation is not being shown, call them v0.2-style fields. `validates: true` means schema-shaped, not legitimate or authorized.
6. Name the permission boundary: public ecological demo content only.
7. Name the refusal boundary: protected or authority-bound content stays out.
8. Read the acceptance/refusal summary.
9. Name what was not run: gateway, full AI runtime, Waka bridge, knowledge commons backend.

## Closing Readout

Use this closing language:

```text
This Open Kit run showed one public ecological commitment entering a small demonstration pool and producing a receipt record. We witnessed what fit, what was partial, and what remains outside this run. The receipt records contribution and review state; it does not establish legitimacy, authority, or certification. TELUS/Jupyter and the larger AI-assisted review flow remain optional next steps, not conditions for this bundle to matter.
```

## Stop Conditions

Pause the demo if:

- protected or authority-bound content is introduced
- someone asks the system to judge cultural legitimacy
- the receipt is being treated as permission or certification
- the demo is being described as a production pool
- participants cannot tell what is being witnessed
- TELUS/Jupyter failure starts distracting from the manual witness surface

If a stop condition fires, name it as a witnessed finding, not as a hidden failure.
